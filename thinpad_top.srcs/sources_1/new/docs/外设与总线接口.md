# RV32I CPU 外设与总线接口详解

## 1. 总线架构概述

本CPU系统采用Wishbone Classic总线协议作为系统互连标准，通过一个4端口交叉开关(wb_mux_4)连接CPU主设备与多个从设备。Wishbone总线是一种开源的片上总线标准，具有简单、灵活的特点。

### 1.1 Wishbone总线信号

Wishbone Classic总线使用以下主要信号：

```systemverilog
// 主设备接口
output logic [31:0] wbm_adr_o;    // 地址输出
output logic [31:0] wbm_dat_o;    // 数据输出
input  wire [31:0] wbm_dat_i;     // 数据输入
output logic        wbm_we_o;      // 写使能
output logic [3:0]  wbm_sel_o;     // 字节选择
output logic        wbm_cyc_o;     // 周期信号
output logic        wbm_stb_o;     // 选通信号
input  wire        wbm_ack_i;     // 应答信号
input  wire        wbm_err_i;     // 错误信号
```

### 1.2 总线时序

Wishbone总线支持多种时序模式，本设计使用标准模式：

- **读操作**：地址有效→选通有效→从设备返回数据→应答有效
- **写操作**：地址有效→数据有效→选通有效→从设备应答

## 2. Wishbone交叉开关

### 2.1 交叉开关结构

4端口交叉开关(wb_mux_4)负责将CPU的访问请求路由到正确的从设备：

```systemverilog
module wb_mux_4 (
    // 主设备接口（CPU）
    input  wire [ADDR_WIDTH-1:0]   wbm_adr_i,
    input  wire [DATA_WIDTH-1:0]   wbm_dat_i,
    output wire [DATA_WIDTH-1:0]   wbm_dat_o,
    input  wire                    wbm_we_i,
    input  wire [SELECT_WIDTH-1:0] wbm_sel_i,
    input  wire                    wbm_stb_i,
    output wire                    wbm_ack_o,
    input  wire                    wbm_cyc_i,
    
    // 从设备0接口（BaseRAM）
    // 从设备1接口（ExtRAM）
    // 从设备2接口（UART）
    // 从设备3接口（CLINT）
    
    // 地址配置
    input  wire [ADDR_WIDTH-1:0]   wbs0_addr,     // BaseRAM地址前缀
    input  wire [ADDR_WIDTH-1:0]   wbs0_addr_msk, // BaseRAM地址掩码
    // ... 其他从设备地址配置
);
```

### 2.2 地址解码与路由

交叉开关根据地址范围将请求路由到对应从设备：

```systemverilog
// 地址匹配检测
wire wbs0_match = ~|((wbm_adr_i ^ wbs0_addr) & wbs0_addr_msk);
wire wbs1_match = ~|((wbm_adr_i ^ wbs1_addr) & wbs1_addr_msk);
wire wbs2_match = ~|((wbm_adr_i ^ wbs2_addr) & wbs2_addr_msk);
wire wbs3_match = ~|((wbm_adr_i ^ wbs3_addr) & wbs3_addr_msk);

// 从设备选择逻辑（优先级从高到低）
wire wbs0_sel = wbs0_match;
wire wbs1_sel = wbs1_match & ~(wbs0_match);
wire wbs2_sel = wbs2_match & ~(wbs0_match | wbs1_match);
wire wbs3_sel = wbs3_match & ~(wbs0_match | wbs1_match | wbs2_match);
```

### 2.3 数据通路

交叉开关根据选择信号将数据路由到正确的从设备：

```systemverilog
// 主设备数据输入（从设备数据输出）
assign wbm_dat_o = wbs0_sel ? wbs0_dat_i :
                   wbs1_sel ? wbs1_dat_i :
                   wbs2_sel ? wbs2_dat_i :
                   wbs3_sel ? wbs3_dat_i :
                   {DATA_WIDTH{1'b0}};

// 主设备应答输入（从设备应答输出）
assign wbm_ack_o = wbs0_ack_i |
                   wbs1_ack_i |
                   wbs2_ack_i |
                   wbs3_ack_i;

// 错误处理
wire select_error = ~(wbs0_sel | wbs1_sel | wbs2_sel | wbs3_sel) & master_cycle;
assign wbm_err_o = wbs0_err_i |
                   wbs1_err_i |
                   wbs2_err_i |
                   wbs3_err_i |
                   select_error;
```

## 3. 内存系统

### 3.1 内存映射

系统采用统一的32位地址空间，具体映射如下：

| 地址范围 | 设备 | 大小 | 描述 |
|---------|------|------|------|
| 0x8000_0000 - 0x803F_FFFF | BaseRAM | 4MB | 基础RAM |
| 0x8040_0000 - 0x807F_FFFF | ExtRAM | 4MB | 扩展RAM |
| 0x1000_0000 - 0x1000_FFFF | UART | 64KB | 串口控制器 |
| 0x0200_0000 - 0x0200_FFFF | CLINT | 64KB | 核心本地中断器 |

### 3.2 SRAM控制器

SRAM控制器实现了Wishbone从设备接口，连接CPU与物理SRAM芯片：

```systemverilog
module sram_controller #(
    parameter SRAM_ADDR_WIDTH = 20,
    parameter SRAM_DATA_WIDTH = 32
) (
    // Wishbone从设备接口
    input wire wb_cyc_i,
    input wire wb_stb_i,
    output reg wb_ack_o,
    input wire [ADDR_WIDTH-1:0] wb_adr_i,
    input wire [DATA_WIDTH-1:0] wb_dat_i,
    output reg [DATA_WIDTH-1:0] wb_dat_o,
    input wire [DATA_WIDTH/8-1:0] wb_sel_i,
    input wire wb_we_i,
    
    // SRAM物理接口
    output reg [SRAM_ADDR_WIDTH-1:0] sram_addr,
    inout wire [SRAM_DATA_WIDTH-1:0] sram_data,
    output reg sram_ce_n,    // 片选，低有效
    output reg sram_oe_n,    // 读使能，低有效
    output reg sram_we_n,    // 写使能，低有效
    output reg [SRAM_BYTES-1:0] sram_be_n  // 字节使能
);
```

### 3.3 SRAM控制器状态机

SRAM控制器使用状态机处理多周期读写操作：

```systemverilog
typedef enum logic [2:0] { 
    IDLE = 0,
    READ = 1,
    READ_2 = 2,
    WRITE = 3,
    WRITE_2 = 4,
    WRITE_3 = 5,
    DONE = 6
} States;

States state;

always_ff @(posedge clk_i or posedge rst_i) begin
    if (rst_i) begin
        state <= IDLE;
        // 初始化信号
    end else begin
        case (state)
            IDLE: begin
                if (wb_stb_i && wb_cyc_i) begin
                    sram_ce_n <= 0;
                    sram_addr <= wb_adr_i[SRAM_ADDR_WIDTH + 1 : 2];
                    sram_be_n <= ~wb_sel_i;
                    if (wb_we_i) begin
                        // 写操作
                        sram_we_n <= 1;
                        sram_data_o <= wb_dat_i;
                        sram_data_t <= 0;
                        state <= WRITE;
                    end else begin
                        // 读操作
                        sram_data_t <= 1;
                        sram_oe_n <= 0;
                        state <= READ;
                    end 
                end
            end 
            READ: begin
                state <= READ_2;
            end 
            READ_2: begin
                sram_ce_n <= 1;
                sram_oe_n <= 1;
                wb_dat_o <= sram_data_i;
                wb_ack_o <= 1;
                state <= DONE;
            end
            WRITE: begin
                sram_we_n <= 0;
                state <= WRITE_2;
            end
            WRITE_2: begin
                sram_we_n <= 1;
                state <= WRITE_3;
            end
            WRITE_3: begin
                sram_ce_n <= 1;
                sram_data_t <= 1;
                wb_ack_o <= 1;
                state <= DONE;
            end
            DONE: begin
                wb_ack_o <= 0;
                state <= IDLE;
            end 
        endcase
    end
end
```

## 4. UART控制器

### 4.1 UART控制器概述

UART控制器提供串行通信功能，支持可配置的波特率和中断：

```systemverilog
module uart_controller #(
    parameter CLK_FREQ = 50_000_000,
    parameter BAUD = 115200
) (
    // Wishbone从设备接口
    input wire wb_cyc_i,
    input wire wb_stb_i,
    output reg wb_ack_o,
    input wire [ADDR_WIDTH-1:0] wb_adr_i,
    input wire [DATA_WIDTH-1:0] wb_dat_i,
    output reg [DATA_WIDTH-1:0] wb_dat_o,
    input wire [DATA_WIDTH/8-1:0] wb_sel_i,
    input wire wb_we_i,
    
    // UART物理接口
    output reg uart_txd_o,
    input  wire uart_rxd_i,
    
    // 状态输出
    output wire tx_ready_o,
    output wire uart_irq
);
```

### 4.2 UART寄存器映射

UART控制器提供两个主要寄存器：

```systemverilog
localparam REG_DATA = 8'h00;    // 数据寄存器
localparam REG_STATUS = 8'h05;   // 状态寄存器
```

### 4.3 UART发送器

UART发送器将并行数据转换为串行数据流：

```systemverilog
// uart transmitter
logic txd_start;
logic txd_busy;
logic [7:0] txd_data;

async_transmitter #(
    .ClkFrequency(CLK_FREQ),
    .Baud        (BAUD)
) u_async_transmitter (
    .clk      (clk_i),
    .TxD_start(txd_start),
    .TxD_data (txd_data),
    .TxD      (uart_txd_o),
    .TxD_busy (txd_busy)
);
```

### 4.4 UART接收器

UART接收器将串行数据流转换为并行数据：

```systemverilog
// uart receiver
logic rxd_data_ready;
logic [7:0] rxd_data;
logic rxd_clear;

async_receiver #(
    .ClkFrequency(CLK_FREQ),
    .Baud        (BAUD)
) u_async_receiver (
    .clk           (clk_i),
    .RxD           (uart_rxd_i),
    .RxD_data_ready(rxd_data_ready),
    .RxD_clear     (rxd_clear),
    .RxD_data      (rxd_data)
);
```

### 4.5 UART状态寄存器

状态寄存器提供发送和接收状态：

```systemverilog
wire [7:0] reg_status = {2'b0, ~txd_busy, 4'b0, rxd_data_ready};
assign tx_ready_o = ~txd_busy;
assign uart_irq = rxd_data_ready;
```

### 4.6 UART Wishbone接口

UART控制器的Wishbone接口实现：

```systemverilog
// 写操作
always_ff @(posedge clk_i) begin
    if (rst_i) begin
        txd_start <= 0;
    end else if(wb_stb_i && wb_we_i) begin
        case (wb_adr_i[7:0])
            REG_DATA: begin
                if(wb_sel_i[0]) begin
                    txd_data  <= wb_dat_i[7:0];
                    txd_start <= 1;
                end else begin
                    txd_start <= 0;
                end
            end
            default: txd_start <= 0;
        endcase
    end else begin
        txd_start <= 0;
    end
end

// 读操作
always_ff @(posedge clk_i) begin
    if(rst_i) begin
        rxd_clear <= 1;
        wb_dat_o <= 32'h0;
    end else if(wb_stb_i && !wb_we_i) begin
        case (wb_adr_i[7:0])
            REG_DATA: begin
                if (wb_sel_i[0]) wb_dat_o[7:0] <= rxd_data;
                if (wb_sel_i[1]) wb_dat_o[15:8] <= rxd_data;
                if (wb_sel_i[2]) wb_dat_o[23:16] <= rxd_data;
                if (wb_sel_i[3]) wb_dat_o[31:24] <= rxd_data;
                rxd_clear <= 1;
            end
            REG_STATUS: begin
                if (wb_sel_i[0]) wb_dat_o[7:0] <= reg_status;
                if (wb_sel_i[1]) wb_dat_o[15:8] <= reg_status;
                if (wb_sel_i[2]) wb_dat_o[23:16] <= reg_status;
                if (wb_sel_i[3]) wb_dat_o[31:24] <= reg_status;
                rxd_clear <= 0;
            end
            default: begin
                rxd_clear <= 0;
            end
        endcase
    end else begin
        rxd_clear <= 0;
    end
end
```

## 5. CLINT（核心本地中断器）

### 5.1 CLINT概述

CLINT提供定时器中断功能，是RISC-V标准的中断控制器：

```systemverilog
module clint #(
    parameter ADDR_WIDTH = 32,
    parameter DATA_WIDTH = 32
) (
    // Wishbone从设备接口
    input wire wb_cyc_i,
    input wire wb_stb_i,
    output reg wb_ack_o,
    input wire [ADDR_WIDTH-1:0] wb_adr_i,
    input wire [DATA_WIDTH-1:0] wb_dat_i,
    output reg [DATA_WIDTH-1:0] wb_dat_o,
    input wire [DATA_WIDTH/8-1:0] wb_sel_i,
    input wire wb_we_i,
    
    // 中断输出
    output reg timer_irq_o
);
```

### 5.2 CLINT寄存器

CLINT提供两个主要寄存器：

- **mtime**：64位定时器计数器
- **mtimecmp**：64位定时器比较寄存器

### 5.3 定时器实现

定时器使用预分频器实现1MHz的计数频率：

```systemverilog
logic [63:0] mtime;
logic [63:0] mtimecmp;
logic [31:0] prescaler;

always_ff @(posedge clk_i) begin
    if (rst_i) begin
        mtime <= 64'd0;
        prescaler <= 32'd0;
    end else begin
        if (prescaler >= 32'd49) begin  // 50MHz/50 = 1MHz
            mtime <= mtime + 64'd1;
            prescaler <= 32'd0;
        end else begin
            prescaler <= prescaler + 32'd1;
        end
    end
end
```

### 5.4 中断生成

当mtime大于等于mtimecmp时，产生中断：

```systemverilog
always_ff @(posedge clk_i) begin
    if (rst_i) begin
        timer_irq_o <= 1'b0;
    end else begin
        if (mtime >= mtimecmp) begin
            timer_irq_o <= 1'b1;
        end else begin
            timer_irq_o <= 1'b0;
        end
    end
end
```

### 5.5 CLINT寄存器访问

CLINT寄存器映射：

```systemverilog
always_comb begin
    wb_dat_o = 32'h0;
    case (wb_adr_i[15:0])
        16'h4000: wb_dat_o = mtimecmp[31:0];    // mtimecmp低32位
        16'h4004: wb_dat_o = mtimecmp[63:32];   // mtimecmp高32位
        16'hBFF8: wb_dat_o = mtime[31:0];       // mtime低32位
        16'hBFFC: wb_dat_o = mtime[63:32];      // mtime高32位
        default:  wb_dat_o = 32'h0;
    endcase
end
```

## 6. 总线仲裁与冲突处理

### 6.1 结构冲突

由于CPU只有一个Wishbone主接口，但需要同时支持指令获取和数据访问，存在结构冲突：

- **冲突场景**：IF阶段取指和MEM阶段访存同时需要总线
- **解决方案**：优先级仲裁，数据访问优先于指令获取

### 6.2 仲裁策略

总线仲裁在CPU核心的状态机中实现：

```systemverilog
always_ff @(posedge clk or posedge rst) begin
    // ...
    case (bus_state)
        BUS_IDLE: begin
            if (mem_request_active) begin
                // 数据访问优先
                bus_state <= BUS_MEM;
                // 设置数据访问参数
            end else if (!fetch_inflight && !fetch_data_valid) begin
                // 指令获取
                bus_state <= BUS_IFETCH;
                // 设置取指参数
            end
        end
        // ... 其他状态
    endcase
end
```

### 6.3 性能影响

结构冲突会导致性能损失：

- **数据访问期间**：指令获取被暂停
- **影响程度**：取决于内存访问频率和延迟
- **缓解措施**：指令缓存可以减少取指频率

## 7. 外设扩展接口

### 7.1 未使用的外设

thinpad_top模块中包含但未使用的外设接口：

- **Flash存储器接口**：用于大容量非易失性存储
- **USB控制器接口**：用于USB设备连接
- **网络控制器接口**：用于以太网连接
- **VGA输出接口**：用于视频输出

### 7.2 扩展方法

添加新外设的步骤：

1. **设计外设Wishbone从接口**
2. **修改交叉开关**：增加端口或替换现有端口
3. **更新地址映射**：为新外设分配地址空间
4. **添加驱动程序**：在操作系统中添加支持

### 7.3 扩展示例

添加新外设到交叉开关：

```systemverilog
// 在thinpad_top中
wb_mux_4 wb_crossbar (
    // 主设备接口
    .wbm_adr_i(cpu_wb_adr),
    // ...
    
    // 从设备0：BaseRAM
    .wbs0_addr(32'h8000_0000),
    .wbs0_addr_msk(32'hFFC0_0000),
    // ...
    
    // 从设备1：ExtRAM
    .wbs1_addr(32'h8040_0000),
    .wbs1_addr_msk(32'hFFC0_0000),
    // ...
    
    // 从设备2：UART
    .wbs2_addr(32'h1000_0000),
    .wbs2_addr_msk(32'hFFFF_0000),
    // ...
    
    // 从设备3：新外设
    .wbs3_addr(32'h0300_0000),
    .wbs3_addr_msk(32'hFFFF_0000),
    // ...
);
```

## 8. 调试与验证

### 8.1 总线监控

可以通过以下方式监控总线活动：

- **总线状态机状态**：观察当前总线操作
- **地址和数据信号**：监控传输内容
- **应答和错误信号**：检查传输状态

### 8.2 外设测试

测试外设功能的方法：

- **寄存器访问测试**：读写外设寄存器
- **功能测试**：验证外设特定功能
- **中断测试**：验证中断生成和处理

### 8.3 常见问题

- **地址冲突**：多个外设使用相同地址范围
- **时序问题**：外设响应时间过长
- **总线错误**：访问未映射地址区域
