# RV32I CPU 流水线设计详解

## 1. 流水线概述

本CPU采用经典的五级流水线结构，包括取指(IF)、译码(ID)、执行(EX)、访存(MEM)和写回(WB)五个阶段。每个阶段在一个时钟周期内完成，实现指令的并行处理，提高处理器吞吐率。

### 1.1 流水线结构图

```
          ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
指令 N    │     IF      │    │     ID      │    │     EX      │    │     MEM     │    │     WB      │
          └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
指令 N+1  │     IF      │    │     ID      │    │     EX      │    │     MEM     │    │     WB      │
          └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
指令 N+2  │     IF      │    │     ID      │    │     EX      │    │     MEM     │    │     WB      │
          └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
指令 N+3  │     IF      │    │     ID      │    │     EX      │    │     MEM     │    │     WB      │
          └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
指令 N+4  │     IF      │    │     ID      │    │     EX      │    │     MEM     │    │     WB      │
          └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 1.2 流水线寄存器

流水线各阶段通过结构化的寄存器进行数据传递：

- **IF/ID寄存器** (`if_id_reg_t`)：传递指令和PC信息
- **ID/EX寄存器** (`id_ex_reg_t`)：传递控制信号和操作数
- **EX/MEM寄存器** (`ex_mem_reg_t`)：传递ALU结果和内存访问信息
- **MEM/WB寄存器** (`mem_wb_reg_t`)：传递写回数据

## 2. 流水线各阶段详解

### 2.1 取指阶段 (IF - Instruction Fetch)

**功能**：从内存中读取指令，并计算下一条指令的PC。

**主要组件**：
- PC寄存器 (`pc_reg`)
- 指令缓存 (I-Cache)
- 分支目标缓冲器 (BTB)
- Wishbone总线接口

**关键操作**：
1. 从PC寄存器获取当前指令地址
2. 检查I-Cache是否命中
3. 若命中，直接从缓存读取指令
4. 若未命中，通过Wishbone总线从内存读取
5. 更新BTB预测信息
6. 计算下一条指令的PC（考虑分支预测）

**状态机**：
- `BUS_IDLE`：总线空闲状态
- `BUS_IFETCH`：指令获取状态
- `BUS_WALK_L1`/`BUS_WALK_L2`：页表遍历状态（虚拟内存支持）

**输出到ID阶段的数据**：
```systemverilog
typedef struct packed {
    logic        valid;      // 指令有效标志
    logic [31:0] pc;        // 当前指令地址
    logic [31:0] instr;     // 指令内容
    logic        pred_taken;    // 分支预测结果
    logic [31:0] pred_target;   // 预测的跳转目标
} if_id_reg_t;
```

### 2.2 译码阶段 (ID - Instruction Decode)

**功能**：解析指令，读取寄存器堆，生成立即数，并进行冲突检测。

**主要组件**：
- 指令译码器
- 寄存器堆 (`rv32i_regfile`)
- 立即数生成器
- 冲突检测单元

**关键操作**：
1. 解析指令的opcode、funct3、funct7字段
2. 生成各类控制信号（ALU操作、内存访问、寄存器写回等）
3. 从寄存器堆读取源操作数
4. 根据指令类型生成立即数
5. 检测Load-Use冲突
6. 处理CSR指令解码

**控制信号生成**：
```systemverilog
// ALU操作类型
typedef enum logic [5:0] {
    ALU_ADD, ALU_SUB, ALU_SLL, ALU_SLT, ALU_SLTU,
    ALU_XOR, ALU_SRL, ALU_SRA, ALU_OR, ALU_AND,
    ALU_PASS_B, ALU_MINU, ALU_XNOR, ALU_CTZ
} alu_op_e;

// 分支类型
typedef enum logic [2:0] {
    BR_NONE, BR_EQ, BR_NE, BR_LT, BR_GE, BR_LTU, BR_GEU
} branch_e;
```

**输出到EX阶段的数据**：
```systemverilog
typedef struct packed {
    logic        valid;
    logic [31:0] pc;
    logic [31:0] instr;
    logic [31:0] imm;           // 立即数
    logic [4:0]  rd, rs1, rs2;  // 寄存器索引
    logic [31:0] rs1_value, rs2_value;  // 寄存器值
    op_a_sel_e   op_a_sel;      // 操作数A选择
    op_b_sel_e   op_b_sel;      // 操作数B选择
    alu_op_e     alu_op;        // ALU操作
    branch_e     branch;        // 分支类型
    logic        jump, jalr;     // 跳转标志
    logic        mem_read, mem_write;  // 内存访问标志
    mem_size_e   mem_size;      // 内存访问大小
    logic        mem_unsigned;  // 无符号访问标志
    logic        reg_write;     // 寄存器写使能
    wb_src_e     wb_src;        // 写回数据源
    // ... 其他控制信号
} id_ex_reg_t;
```

### 2.3 执行阶段 (EX - Execute)

**功能**：执行算术逻辑运算，计算分支目标，处理前递。

**主要组件**：
- ALU（算术逻辑单元）
- 前递单元
- 分支判断逻辑
- CSR写逻辑

**关键操作**：
1. 根据前递单元选择操作数
2. 执行ALU运算（算术、逻辑、移位等）
3. 计算内存访问地址
4. 判断分支条件
5. 计算跳转目标地址
6. 准备CSR写数据

**前递逻辑**：
```systemverilog
always_comb begin
    // 默认使用ID阶段读取的寄存器值
    ex_rs1_value = id_ex_reg.rs1_value;
    ex_rs2_value = id_ex_reg.rs2_value;
    
    // EX/MEM阶段前递（优先级高）
    if (id_ex_reg.use_rs1 && ex_mem_reg.valid && ex_mem_reg.reg_write && 
        (ex_mem_reg.rd != 5'd0) && (ex_mem_reg.rd == id_ex_reg.rs1)) begin
        ex_rs1_value = ex_mem_forward_value;
    end
    
    // MEM/WB阶段前递
    if (id_ex_reg.use_rs1 && mem_wb_reg.valid && mem_wb_reg.reg_write && 
        (mem_wb_reg.rd != 5'd0) && (mem_wb_reg.rd == id_ex_reg.rs1)) begin
        ex_rs1_value = mem_wb_forward_value;
    end
    // 类似处理rs2...
end
```

**ALU操作**：
```systemverilog
always_comb begin
    case (id_ex_reg.alu_op)
        ALU_ADD:   alu_result = alu_operand_a + alu_operand_b;
        ALU_SUB:   alu_result = alu_operand_a - alu_operand_b;
        ALU_SLL:   alu_result = alu_operand_a << alu_operand_b[4:0];
        ALU_SLT:   alu_result = ($signed(alu_operand_a) < $signed(alu_operand_b)) ? 32'd1 : 32'd0;
        ALU_SLTU:  alu_result = (alu_operand_a < alu_operand_b) ? 32'd1 : 32'd0;
        ALU_XOR:   alu_result = alu_operand_a ^ alu_operand_b;
        ALU_XNOR:  alu_result = ~(alu_operand_a ^ alu_operand_b);
        ALU_SRL:   alu_result = alu_operand_a >> alu_operand_b[4:0];
        ALU_SRA:   alu_result = $signed(alu_operand_a) >>> alu_operand_b[4:0];
        ALU_OR:    alu_result = alu_operand_a | alu_operand_b;
        ALU_AND:   alu_result = alu_operand_a & alu_operand_b;
        ALU_PASS_B:alu_result = alu_operand_b;
        ALU_MINU:  alu_result = (alu_operand_a < alu_operand_b) ? alu_operand_a : alu_operand_b;
        ALU_CTZ:   alu_result = ctz32(ex_rs1_value);
        default:   alu_result = 32'h0;
    endcase
end
```

**输出到MEM阶段的数据**：
```systemverilog
typedef struct packed {
    logic        valid;
    logic [31:0] pc, pc_plus4;
    logic [4:0]  rd;
    logic        reg_write;
    wb_src_e     wb_src;
    logic        mem_read, mem_write;
    mem_size_e   mem_size;
    logic        mem_unsigned;
    logic [31:0] alu_result;     // ALU结果/内存地址
    logic [31:0] rs2_value;      // 存储数据
    logic [31:0] branch_target;  // 分支目标
    logic        branch_taken;   // 分支是否跳转
    logic        jump, jalr;      // 跳转标志
    // ... 其他信号
} ex_mem_reg_t;
```

### 2.4 访存阶段 (MEM - Memory Access)

**功能**：读写数据内存，处理内存访问异常。

**主要组件**：
- Wishbone总线接口
- 数据TLB
- 对齐逻辑
- 异常检测

**关键操作**：
1. 检查内存访问权限（虚拟内存支持）
2. 处理地址对齐
3. 发起Wishbone总线事务
4. 处理字节/半字/字访问
5. 检测内存访问异常

**内存访问状态机**：
```systemverilog
typedef enum logic [2:0] {
    BUS_IDLE,
    BUS_IFETCH,
    BUS_MEM,
    BUS_WALK_L1,
    BUS_WALK_L2
} bus_state_e;
```

**数据对齐处理**：
```systemverilog
always_comb begin
    store_sel_comb  = 4'b0000;
    store_data_comb = 32'h0;
    case (id_ex_reg.mem_size)
        MEM_SIZE_BYTE: begin
            store_sel_comb  = 4'b0001 << alu_result[1:0];
            store_data_comb = {4{ex_rs2_value[7:0]}} << (8 * alu_result[1:0]);
        end
        MEM_SIZE_HALF: begin
            store_sel_comb  = alu_result[1] ? 4'b1100 : 4'b0011;
            store_data_comb = {2{ex_rs2_value[15:0]}} << (16 * alu_result[1]);
        end
        MEM_SIZE_WORD: begin
            store_sel_comb  = 4'b1111;
            store_data_comb = ex_rs2_value;
        end
    endcase
end
```

**输出到WB阶段的数据**：
```systemverilog
typedef struct packed {
    logic        valid;
    logic [4:0]  rd;
    logic        reg_write;
    wb_src_e     wb_src;
    logic [31:0] alu_result;  // ALU结果
    logic [31:0] mem_data;    // 内存读取数据
    logic [31:0] pc_plus4;    // PC+4（用于跳转指令）
    logic [31:0] csr_data;    // CSR读取数据
} mem_wb_reg_t;
```

### 2.5 写回阶段 (WB - Write Back)

**功能**：将结果写回寄存器堆。

**主要组件**：
- 写回多路选择器
- 寄存器堆写端口

**关键操作**：
1. 根据wb_src选择写回数据源
2. 将数据写回寄存器堆
3. 处理x0寄存器特殊情况

**写回数据选择**：
```systemverilog
always_comb begin
    case (mem_wb_reg.wb_src)
        WB_SRC_MEM: regfile_wdata = mem_wb_reg.mem_data;
        WB_SRC_PC4: regfile_wdata = mem_wb_reg.pc_plus4;
        WB_SRC_CSR: regfile_wdata = mem_wb_reg.csr_data;
        default:    regfile_wdata = mem_wb_reg.alu_result;
    endcase
end
```

## 3. 流水线冲突处理

### 3.1 结构冲突

**问题**：取指(IF)和访存(MEM)都需要访问Wishbone总线，但只有一个总线接口。

**解决方案**：
- 优先级：数据访问 > 指令获取
- 当MEM阶段需要访问内存时，暂停取指
- 通过总线状态机协调访问顺序

### 3.2 数据冲突

**Load-Use冲突**：
- **检测**：在ID阶段检测前一条指令是否为Load且目标寄存器被当前指令使用
- **解决**：插入气泡，停顿流水线一个周期

```systemverilog
assign load_use_hazard = id_ex_reg.valid && id_ex_reg.mem_read && (id_ex_reg.rd != 5'd0) &&
                         ((id_use_rs1 && (id_ex_reg.rd == if_id_reg.instr[19:15])) ||
                          (id_use_rs2 && (id_ex_reg.rd == if_id_reg.instr[24:20])));
```

**一般数据冲突**：
- **解决**：通过前递(Forwarding)机制
- **优先级**：EX/MEM阶段 > MEM/WB阶段

### 3.3 控制冲突

**问题**：分支指令在EX阶段才确定是否跳转，导致已取指的后续指令可能需要丢弃。

**解决方案**：
- **分支预测**：使用BTB进行预测
- **预测错误处理**：检测预测错误时冲刷流水线
- **Misprediction检测**：比较实际跳转结果与预测结果

```systemverilog
always_comb begin
    branch_mispredicted = 1'b0;
    if (id_ex_reg.valid) begin
        if (id_ex_reg.branch != BR_NONE || id_ex_reg.jump) begin
            if (branch_taken_ex != id_ex_reg.pred_taken)
                branch_mispredicted = 1'b1;
            else if (branch_taken_ex && (branch_target_ex != id_ex_reg.pred_target))
                branch_mispredicted = 1'b1;
        end else begin
            if (id_ex_reg.pred_taken)
                branch_mispredicted = 1'b1;
        end
    end
end
```

## 4. 流水线优化技术

### 4.1 分支预测

**BTB设计**：
- 64条目直接映射
- 2位饱和计数器
- 存储目标地址

```systemverilog
typedef struct packed {
    logic        valid;
    logic [BTB_TAG_BITS-1:0] tag;
    logic [31:0] target;
    logic [1:0]  counter;
} btb_entry_t;
```

**预测更新**：
- 跳转发生：计数器增加
- 跳转未发生：计数器减少
- 新条目：初始化为弱不跳转(01)或弱跳转(10)

### 4.2 指令缓存

**I-Cache设计**：
- 128行直接映射
- 32位数据宽度
- 写分配策略
- LRU替换（简单实现）

**缓存操作**：
- 命中：直接提供数据
- 未命中：通过总线获取并填充缓存
- 刷新：FENCE.I指令或SFENCE.VMA指令

### 4.3 前递优化

**多级前递**：
- EX/MEM → EX：高优先级
- MEM/WB → EX：低优先级
- Load结果特殊处理：Load-Use冲突检测

**前递数据源选择**：
```systemverilog
always_comb begin
    // 选择前递数据源
    unique case (ex_mem_reg.wb_src)
        WB_SRC_MEM: ex_mem_forward_value = mem_load_data;
        WB_SRC_PC4: ex_mem_forward_value = ex_mem_reg.pc_plus4;
        WB_SRC_CSR: ex_mem_forward_value = ex_mem_reg.csr_rdata;
        default:    ex_mem_forward_value = ex_mem_reg.alu_result;
    endcase
end
```

## 5. 流水线性能分析

### 5.1 吞吐率

- **理想情况**：每周期完成一条指令
- **分支预测命中**：无性能损失
- **Load-Use冲突**：损失1个周期
- **分支预测错误**：损失2-3个周期
- **内存访问**：阻塞流水线

### 5.2 加速比

相比单周期实现：
- 理想加速比：5倍
- 实际加速比：约3-4倍（考虑冲突和停顿）

### 5.3 效率

- **CPI（每指令周期数）**：理想1.0，实际约1.3-1.5
- **流水线效率**：约70-80%
- **瓶颈**：内存访问延迟和分支预测错误

## 7. 流水线扩展性

### 7.1 支持的扩展

- **B扩展**：MINU、CTZ、XNOR指令
- **M扩展**：乘除法指令（可添加）
- **F扩展**：浮点指令（需添加浮点流水线）
- **C扩展**：压缩指令（需添加译码逻辑）

### 7.2 多级流水线

- 可扩展为更多流水线阶段
- 增加更复杂的分支预测
- 实现乱序执行（需大幅修改）

### 7.3 多核支持

- 添加缓存一致性协议
- 实现多核仲裁
- 支持原子操作
