# RV32I CPU 异常与中断处理详解

## 1. 异常与中断概述

本CPU实现了RISC-V标准的异常和中断处理机制，支持机器模式(M-mode)下的异常处理和中断响应。系统可以处理多种类型的异常，包括非法指令、内存访问错误、对齐错误等，以及来自外设的中断请求。

### 1.1 异常与中断的区别

- **异常(Exception)**：由CPU内部事件引起，如非法指令、内存访问错误等
- **中断(Interrupt)**：由外部事件引起，如定时器中断、外部中断等

### 1.2 RISC-V异常处理模型

RISC-V采用统一的异常处理模型，通过以下CSR寄存器管理异常：

- **mstatus**：机器状态寄存器，控制全局中断使能
- **mtvec**：机器陷阱向量寄存器，定义异常处理程序入口
- **mepc**：机器异常程序计数器，保存发生异常时的PC
- **mcause**：机器异常原因寄存器，记录异常类型和原因
- **mtval**：机器陷阱值寄存器，保存与异常相关的额外信息
- **mie**：机器中断使能寄存器，控制各类中断的使能
- **mip**：机器中断挂起寄存器，显示当前挂起的中断

## 2. CSR寄存器详解

### 2.1 mstatus寄存器

mstatus寄存器控制处理器的全局状态和中断使能：

```systemverilog
logic [31:0] csr_mstatus;

// 关键字段
// [3]  MIE: 机器中断使能
// [7]  MPIE: 机器中断使能之前的值
// [12:11] MPP: 机器前一特权级
```

### 2.2 mtvec寄存器

mtvec寄存器定义异常处理程序的入口地址：

```systemverilog
logic [31:0] csr_mtvec;

// 字段定义
// [31:2] BASE: 异常处理程序基地址
// [1:0] MODE: 异常处理模式
//   00 = 直接模式：所有异常跳转到BASE
//   01 = 向量模式：根据异常原因跳转不同入口（本实现仅支持直接模式）
```

### 2.3 mepc寄存器

mepc寄存器保存发生异常时的PC值：

```systemverilog
logic [31:0] csr_mepc;

// 当异常发生时，当前PC（或异常指令地址）保存到mepc
// mret指令使用mepc恢复执行
```

### 2.4 mcause寄存器

mcause寄存器记录异常的类型和原因：

```systemverilog
logic [31:0] csr_mcause;

// 字段定义
// [31]    Interrupt: 1表示中断，0表示异常
// [30:0]  Exception Code: 异常/中断代码
```

### 2.5 mtval寄存器

mtval寄存器保存与异常相关的额外信息：

```systemverilog
logic [31:0] csr_mtval;

// 对于地址异常，保存出错地址
// 对于非法指令，保存指令本身
// 其他异常可能为0
```

### 2.6 mie和mip寄存器

mie和mip寄存器控制中断使能和挂起状态：

```systemverilog
logic [31:0] csr_mie;  // 中断使能
logic [31:0] csr_mip;  // 中断挂起

// 关键中断位
// [11] MEIE: 机器外部中断使能/挂起
// [7]  MTIE: 机器定时器中断使能/挂起
```

## 3. 异常类型与处理

### 3.1 异常类型定义

本CPU支持的异常类型：

```systemverilog
localparam [4:0] CAUSE_INSTR_ADDR_MISALIGNED = 5'd0;
localparam [4:0] CAUSE_INSTR_ACCESS_FAULT    = 5'd1;
localparam [4:0] CAUSE_ILLEGAL_INSTRUCTION   = 5'd2;
localparam [4:0] CAUSE_BREAKPOINT            = 5'd3;
localparam [4:0] CAUSE_LOAD_ADDR_MISALIGNED  = 5'd4;
localparam [4:0] CAUSE_LOAD_ACCESS_FAULT     = 5'd5;
localparam [4:0] CAUSE_STORE_ADDR_MISALIGNED = 5'd6;
localparam [4:0] CAUSE_STORE_ACCESS_FAULT    = 5'd7;
localparam [4:0] CAUSE_ECALL_U               = 5'd8;   // U-mode ECALL
localparam [4:0] CAUSE_ECALL_S               = 5'd9;   // S-mode ECALL
localparam [4:0] CAUSE_ECALL_M               = 5'd11;  // M-mode ECALL
localparam [4:0] CAUSE_INSTR_PAGE_FAULT      = 5'd12;
localparam [4:0] CAUSE_LOAD_PAGE_FAULT       = 5'd13;
localparam [4:0] CAUSE_STORE_PAGE_FAULT      = 5'd15;
```

### 3.2 异常检测逻辑

异常检测在EX阶段进行：

```systemverilog
always_comb begin
    ex_trap_valid = 1'b0;
    ex_trap_cause = CAUSE_ILLEGAL_INSTRUCTION;
    ex_trap_tval  = 32'h0;

    if (id_ex_reg.valid) begin
        if (id_ex_reg.illegal) begin
            ex_trap_valid = 1'b1;
            ex_trap_cause = CAUSE_ILLEGAL_INSTRUCTION;
            ex_trap_tval  = id_ex_reg.instr;
        end else if (id_ex_reg.is_ecall) begin
            ex_trap_valid = 1'b1;
            case (priv_mode)
                PRIV_U: ex_trap_cause = CAUSE_ECALL_U;  // U-mode: mcause = 8
                PRIV_S: ex_trap_cause = CAUSE_ECALL_S;  // S-mode: mcause = 9
                default: ex_trap_cause = CAUSE_ECALL_M; // M-mode: mcause = 11
            endcase
            ex_trap_tval  = 32'h0;
        end else if (id_ex_reg.is_ebreak) begin
            ex_trap_valid = 1'b1;
            ex_trap_cause = CAUSE_BREAKPOINT;
            ex_trap_tval  = id_ex_reg.pc;
        end else if (id_ex_reg.jump && id_ex_reg.jalr && |branch_target_ex[1:0]) begin
            ex_trap_valid = 1'b1;
            ex_trap_cause = CAUSE_INSTR_ADDR_MISALIGNED;
            ex_trap_tval  = branch_target_ex;
        end else if (id_ex_reg.mem_read || id_ex_reg.mem_write) begin
            logic misaligned;
            misaligned = 1'b0;
            case (id_ex_reg.mem_size)
                MEM_SIZE_HALF: misaligned = alu_result[0];
                MEM_SIZE_WORD: misaligned = |alu_result[1:0];
                default: misaligned = 1'b0;
            endcase
            if (misaligned) begin
                ex_trap_valid = 1'b1;
                ex_trap_cause = id_ex_reg.mem_read ? CAUSE_LOAD_ADDR_MISALIGNED : CAUSE_STORE_ADDR_MISALIGNED;
                ex_trap_tval  = alu_result;
            end
        end
    end
end
```

### 3.3 异常处理流程

异常处理分为三个阶段：

1. **检测阶段**：在EX阶段检测异常
2. **等待阶段**：等待流水线清空
3. **处理阶段**：更新CSR，跳转到异常处理程序

```systemverilog
// 异常请求生成
always_comb begin
    trap_request     = 1'b0;
    trap_cause_value = 32'h0;
    trap_tval_value  = 32'h0;
    trap_target_pc   = mtvec_base;
    trap_mepc_value  = 32'h0;

    // 中断优先级高于异常
    if (m_timer_interrupt_pending) begin
        trap_request     = 1'b1;
        trap_cause_value = {1'b1, 26'h0, CAUSE_M_TIMER_INT};
        trap_tval_value  = 32'h0;
        trap_mepc_value  = interrupt_mepc;
    end else if (m_external_interrupt_pending) begin
        trap_request     = 1'b1;
        trap_cause_value = {1'b1, 26'h0, CAUSE_M_EXTERNAL_INT};
        trap_tval_value  = 32'h0;
        trap_mepc_value  = interrupt_mepc;
    end else if (ex_trap_valid) begin
        trap_request     = 1'b1;
        trap_cause_value = {27'h0, ex_trap_cause};
        trap_tval_value  = ex_trap_tval;
        trap_mepc_value = id_ex_reg.pc;
    end else if (instr_bus_trap_valid) begin
        trap_request     = 1'b1;
        trap_cause_value = {27'h0, CAUSE_INSTR_ACCESS_FAULT};
        trap_tval_value  = instr_error_pc;
        trap_mepc_value  = instr_error_pc;
    end else if (mem_bus_trap_valid) begin
        trap_request     = 1'b1;
        trap_cause_value = {27'h0, mem_bus_trap_cause};
        trap_tval_value  = mem_bus_trap_tval;
        trap_mepc_value  = mem_active_pc;
    end
end
```

### 3.4 异常状态机

异常处理使用状态机控制：

```systemverilog
typedef enum logic [1:0] {TRAP_IDLE, TRAP_WAIT, TRAP_FLUSH} trap_state_e;
trap_state_e trap_state;

always_ff @(posedge clk or posedge rst) begin
    if (rst) begin
        trap_state         <= TRAP_IDLE;
        trap_cause_reg     <= 32'h0;
        trap_tval_reg      <= 32'h0;
        trap_target_pc_reg <= 32'h0;
        trap_mepc_reg      <= 32'h0;
    end else begin
        unique case (trap_state)
            TRAP_IDLE: begin
                if (trap_request) begin
                    trap_state         <= TRAP_WAIT;
                    trap_cause_reg     <= trap_cause_value;
                    trap_tval_reg      <= trap_tval_value;
                    trap_target_pc_reg <= trap_target_pc;
                    trap_mepc_reg      <= trap_mepc_value;
                end
            end
            TRAP_WAIT: begin
                trap_state <= TRAP_FLUSH;
            end
            TRAP_FLUSH: begin
                trap_state <= TRAP_IDLE;
            end
            default: trap_state <= TRAP_IDLE;
        endcase
    end
end
```

## 4. 中断处理

### 4.1 中断类型

本CPU支持以下中断类型：

```systemverilog
localparam [4:0] CAUSE_M_TIMER_INT    = 5'd7;   // 机器定时器中断
localparam [4:0] CAUSE_M_EXTERNAL_INT = 5'd11;  // 机器外部中断
```

### 4.2 中断检测逻辑

中断检测在每周期进行：

```systemverilog
wire global_interrupt_enable = (priv_mode == PRIV_M) ? csr_mstatus[3] : 1'b1;

wire m_external_interrupt_pending = global_interrupt_enable && csr_mie[11] && csr_mip[11];
wire m_timer_interrupt_pending    = global_interrupt_enable && csr_mie[7]  && csr_mip[7];
```

### 4.3 中断处理流程

中断处理流程与异常类似，但有以下特点：

1. **中断优先级**：定时器中断和外部中断有固定优先级
2. **中断返回地址**：通常是当前正在执行的指令地址
3. **中断嵌套**：通过MIE/MPIE机制支持

```systemverilog
// 中断返回地址计算
always_comb begin
    // 优先级：EX阶段 > ID阶段 > IF阶段 > 当前PC
    if (id_ex_reg.valid)
        interrupt_mepc = id_ex_reg.pc;
    else if (if_id_reg.valid)
        interrupt_mepc = if_id_reg.pc;
    else if (fetch_data_valid)
        interrupt_mepc = fetch_data_pc;
    else
        interrupt_mepc = pc_reg;
end
```

## 5. CSR访问与更新

### 5.1 CSR访问控制

CSR访问需要检查权限和寄存器合法性：

```systemverilog
function automatic logic csr_legal(input logic [11:0] addr);
    case (addr)
        12'h300, 12'h301, 12'h304, 12'h305,
        12'h306, 12'h340, 12'h341, 12'h342, 
        12'h343, 12'h344, 12'h3A0, 12'h3B0,
        12'hB00, 12'hB02, 12'hB80, 12'hB82,
        12'hC00, 12'hC01, 12'hC02,
        12'hF11, 12'hF12, 12'hF13, 12'hF14,
        12'h180: csr_legal = 1'b1;
        default: csr_legal = 1'b0;
    endcase
endfunction

function automatic logic csr_is_readonly(input logic [11:0] addr);
    case (addr)
        12'h301: csr_is_readonly = 1'b1;  // misa
        12'hF11, 12'hF12, 12'hF13, 12'hF14: csr_is_readonly = 1'b1;  // vendor info
        12'hC00, 12'hC01, 12'hC02: csr_is_readonly = 1'b1;  // cycle/time/instret
        12'hB80, 12'hB82: csr_is_readonly = 1'b1;  // upper counters
        default: csr_is_readonly = 1'b0;
    endcase
endfunction
```

### 5.2 CSR读写操作

CSR读写操作支持三种模式：

```systemverilog
typedef enum logic [1:0] {
    CSR_OP_WRITE,  // CSRRW/CSRRWI：直接写入
    CSR_OP_SET,    // CSRRS/CSRRSI：置位操作
    CSR_OP_CLEAR   // CSRRC/CSRRCI：清位操作
} csr_op_e;
```

### 5.3 CSR状态更新

CSR状态在异常发生时和MRET指令执行时更新：

```systemverilog
always_ff @(posedge clk or posedge rst) begin
    if (rst) begin
        csr_mstatus  <= 32'h0000_0000;
        csr_mtvec    <= RESET_VECTOR;
        csr_mepc     <= RESET_VECTOR;
        csr_mcause   <= 32'h0;
        csr_mtval    <= 32'h0;
        csr_mscratch <= 32'h0;
        csr_mie      <= 32'h0;
        csr_mip      <= 32'h0;
        priv_mode    <= PRIV_M;
    end else begin
        // 更新计数器
        csr_mcycle <= csr_mcycle + 32'd1;
        if (instr_retired)
            csr_minstret <= csr_minstret + 32'd1;
        
        // 更新中断挂起状态
        csr_mip[11] <= external_interrupt_i;
        csr_mip[7]  <= timer_interrupt_i;
        
        if (trap_active) begin
            // 异常处理时更新CSR
            csr_mepc   <= trap_mepc_reg;
            csr_mcause <= trap_cause_reg;
            csr_mtval  <= trap_tval_reg;
            csr_mstatus<= csr_mstatus_trap_value;
            priv_mode  <= PRIV_M;
        end else if (mret_commit) begin
            // MRET指令执行时恢复CSR
            csr_mstatus<= csr_mstatus_mret_value;
            priv_mode  <= mret_target_priv;
        end else if (ex_mem_reg.csr_en && ex_mem_reg.csr_write_en && mem_commit && !ex_mem_reg.trap_valid) begin
            // CSR写操作
            unique case (ex_mem_reg.csr_addr)
                12'h300: csr_mstatus  <= ex_mem_reg.csr_wdata;
                12'h304: csr_mie      <= ex_mem_reg.csr_wdata;
                12'h305: csr_mtvec    <= ex_mem_reg.csr_wdata;
                12'h340: csr_mscratch <= ex_mem_reg.csr_wdata;
                12'h341: csr_mepc     <= ex_mem_reg.csr_wdata;
                12'h342: csr_mcause   <= ex_mem_reg.csr_wdata;
                12'h343: csr_mtval    <= ex_mem_reg.csr_wdata;
                12'h344: csr_mip      <= (ex_mem_reg.csr_wdata & ~32'h880) |
                                       (csr_mip & 32'h880);
                12'hB00: csr_mcycle   <= ex_mem_reg.csr_wdata;
                12'hB02: csr_minstret <= ex_mem_reg.csr_wdata;
                12'h180: csr_satp     <= ex_mem_reg.csr_wdata;
                default: ;
            endcase
        end
    end
end
```

## 7. 流水线刷新与恢复

### 7.1 流水线刷新

当异常发生时，需要刷新流水线：

```systemverilog
// 流水线刷新信号
assign trap_flush  = trap_active;
assign fencei_flush = ex_mem_reg.valid && ex_mem_reg.is_fencei && !ex_mem_reg.trap_valid;
assign mret_flush   = ex_mem_reg.valid && ex_mem_reg.is_mret && !ex_mem_reg.trap_valid;
assign branch_flush = branch_mispredicted && !trap_flush;

// PC更新
always_ff @(posedge clk or posedge rst) begin
    if (rst) begin
        pc_reg <= RESET_VECTOR;
    end else if (trap_flush) begin
        pc_reg <= trap_target_pc_reg;
    end else if (mret_flush) begin
        pc_reg <= csr_mepc;
    end else if (branch_flush) begin
        if (branch_taken_ex)
            pc_reg <= branch_target_ex;
        else
            pc_reg <= id_ex_reg.pc + 32'd4;
    end
end
```

### 7.2 流水线寄存器更新

流水线寄存器在异常时需要清空或保持：

```systemverilog
// IF/ID寄存器更新
always_ff @(posedge clk or posedge rst) begin
    if (rst) begin
        if_id_reg <= '{default: '0};
    end else if (trap_flush || fencei_flush || mret_flush) begin
        if_id_reg.valid <= 1'b0;
    end else if (branch_flush) begin
        if_id_reg.valid <= 1'b0;
    end else if (!stall_id) begin
        // 正常更新
    end
end

// ID/EX寄存器更新
always_ff @(posedge clk or posedge rst) begin
    if (rst) begin
        id_ex_reg <= '0;
    end else if (trap_flush || fencei_flush || mret_flush) begin
        id_ex_reg.valid <= 1'b0;
    end else if (!stall_ex) begin
        // 正常更新
    end
end
```

// 页故障原因

localparam [4:0] CAUSE_LOAD_PAGE_FAULT  = 5'd13;

// 页故障检测

## 10. 性能考虑

### 10.1 异常处理延迟

异常处理需要多个周期：

1. **异常检测**：1个周期（EX阶段）
2. **流水线刷新**：1-2个周期
3. **CSR更新**：1个周期
4. **跳转到处理程序**：1个周期

总计：约4-5个周期

### 10.2 中断延迟

中断延迟取决于中断检测时机：

- **最佳情况**：1-2个周期
- **最坏情况**：取决于当前指令执行时间

### 10.3 优化措施

- **快速中断检测**：在每周期检测中断条件
- **流水线并行处理**：异常处理与正常执行并行
- **缓存CSR**：减少CSR访问延迟
