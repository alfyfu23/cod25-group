test 1 测试说明
本实验内容为 32 位 RISC-V CPU 设计实现。测试脚本将利用监控程序（基础版，关闭 UART16550）对 CPU 功能进行验证。测试对 CPU 实现有如下要求：

内存空间为 0x8000_0000～0x807F_FFFF，共 8MB，整个内存空间均要求可读可写可执只执行。内存空间的 0x8000_0000～0x803F_FFFF 地址映射到 BaseRAM，0x8040_0000～0x807F_FFFF 映射到 ExtRAM。
使用直连串口控制器，可以直接使用实验模板中提供的 Wishbone UART controller，地址映射按照监控程序 README 中的要求。
CPU 时钟使用外部时钟输入，两路时钟输入均可使用，建议使用 PLL 模块升高或降低时钟频率。
CPU 复位后从 0x8000_0000 开始取指令。
测试步骤
监控程序测试
监控程序 Kernel 下载到 BaseRAM 中
单击复位按钮
模拟 Term 程序连接串口，等待 Kernel 启动时的欢迎信息
用 A 命令将用户程序加载到 0x8010_0000 处
用 D 命令读出用户程序，检查是否正确加载
用 G 命令执行用户程序
用 R 和 D 命令读取用户程序执行后的寄存器和内存，检查是否正确执行

一些额外补充要求和信息：
1. 保证指令存储/数据存储地址布局满足：
   - 0x8000_0000：Kernel 代码入口
   - 0x8010_0000~：用户程序区域
   - 0x1000_0000~：串口寄存器
2. 串口遵循简化版 16550 寄存器：0x1000_0000 数据，0x1000_0005 状态位[5][0] 等
